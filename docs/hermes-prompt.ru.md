# Hermes setup prompts

English version: [hermes-prompt.md](hermes-prompt.md)

Этот документ содержит примеры промптов для использования Hermes при установке и настройке `hm-openrouter-monitoring`.

## Prompt 1: установка с подтверждением

Используй этот промпт, когда репозиторий уже клонирован и нужно безопасно установить его в текущее окружение Hermes.

```text
Установи hm-openrouter-monitoring из текущего репозитория в это окружение Hermes.

Перед внесением любых изменений сначала изучи локальную файловую систему и определи:
- правильную директорию скриптов Hermes;
- правильную директорию skills Hermes;
- правильную state-директорию для файлов watchdog;
- настроен ли уже OPENROUTER_API_KEY;
- существуют ли уже cron-задачи, связанные с OpenRouter.

Перед включением любых опциональных возможностей обязательно спроси меня отдельно, в том числе:
- нужна ли конвертация в локальную валюту, например RUB;
- нужен ли daily brief;
- нужна ли доставка в Telegram;
- нужны ли пользовательские пороги;
- нужны ли пользовательские расписания.

Не предполагай пути /opt по умолчанию, если они не существуют и не соответствуют текущей системе.

После анализа покажи:
1. какие пути ты хочешь использовать;
2. какие переменные окружения ты хочешь задать;
3. какие cron-задачи ты хочешь создать или изменить;
4. какие файлы ты хочешь скопировать.

Дождись моего подтверждения перед применением любых изменений.
```

## Prompt 2: адаптация существующей установки

Используй этот промпт, когда мониторинг OpenRouter уже существует и нужно привести его в соответствие с этим репозиторием.

```text
Проведи аудит текущего окружения Hermes и сравни его с репозиторием hm-openrouter-monitoring.

Найди текущие установленные версии:
- openrouter_watchdog.py
- openrouter_anomaly_analyzer.py
- openrouter_daily_brief.py
- связанные skills и cron-задачи

После этого подготовь минимальный план адаптации.

Правила:
- не ломай уже работающую конфигурацию;
- сохраняй инфраструктурные пути, если нет необходимости их менять;
- отдельно выделяй опциональные возможности и обязательные изменения;
- спрашивай отдельно перед включением конвертации в локальную валюту, daily brief или Telegram delivery;
- показывай diff перед записью файлов.

Верни:
1. summary текущего состояния;
2. предлагаемые изменения;
3. список файлов для обновления;
4. вопросы, требующие подтверждения владельца.
```

## Prompt 3: только проверка

Используй этот промпт, когда нужно только проверить установку без внесения изменений.

```text
Проверь текущую установку hm-openrouter-monitoring в этом окружении Hermes.

Проверь:
- расположение скриптов;
- расположение skill;
- state-директорию;
- переменные окружения, используемые скриптами;
- существующие cron-задачи;
- может ли watchdog записывать state-файлы.

Ничего не изменяй.

Верни:
1. найденные пути;
2. найденные переменные окружения;
3. найденные cron-задачи;
4. чего не хватает;
5. какие опциональные возможности уже включены.
```
